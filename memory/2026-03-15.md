## PolyQuant Investigation: Dual/Hedge Trade Logic

**Time**: 5:00 PM CST (March 15)

### Investigation: Does Polytrader execute dual trades?

**User Question**: Whether the system executes Primary Trade (100% Kelly) + Hedge Trade (20% Kelly on opposite side) simultaneously, when hedge was claimed to be only needed for price-maker trades, not price-taker trades.

**Finding**: **YES - Hedge IS implemented in poly-hft-pro**

**Location**: `/root/.openclaw/workspace/openclaw-polytrader/skills/poly-hft-pro`

**Implementation Details**:
- **Hedge ratio**: 12.5% (not 20%) of primary position size
- **Kelly Engine** (`kelly_engine.rs`): Calculates `hedge_size = primary_size * 0.125`
- **Executor** (`executor_task.rs`): Places TWO trades per signal:
  - Primary position: 100% Kelly size (directional bet)
  - Hedge position: 12.5% size (opposite direction)
- **Shared State**: Tracks both `primary_position` and `hedge_position`

**Code Evidence**:
```rust
// kelly_engine.rs line ~258
let hedge_size = primary_size * hedge_ratio;  // 0.125 = 12.5%

// executor_task.rs lines ~200-288
shared_state.set_position(Some(primary), Some(hedge)).await;
```

**User Confirmation**: Hedge is NOT needed for price-taker trades.

**Pending Action**: Removal plan drafted - need to remove hedge from:
1. `executor_task.rs` - hedge position creation
2. `kelly_engine.rs` - hedge_size calculations
3. `gen2_trading.rs` - hedge handling
4. `config.rs` - set hedge_ratio to 0.0

**Note**: PolyQuant (the Python/dashboard version) does NOT have hedge logic - only single trades. The hedge is only in the Rust poly-hft-pro implementation.
