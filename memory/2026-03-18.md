## KimiClaw Mode Added (01:55)

### User Request
User asked to use the existing Kimi K2.5 connection (KimiClaw) instead of requiring external API keys.

### Implementation

**Created Files:**
- `kimiclaw_submit.py` - Tool to submit trading decisions from KimiClaw
- `kimiclaw_monitor.py` - Monitor script for pending requests

**Modified Files:**
- `llm_decision_engine.py` - Added `_call_kimiclaw()` method for file-based communication
- `LLM_WORKFLOW.md` - Added KimiClaw mode documentation

### How KimiClaw Mode Works

1. **Set provider:** `export LLM_PROVIDER=kimiclaw`
2. **Engine generates prompt** with market data → writes to `kimiclaw_request.json`
3. **User tells KimiClaw** the market context
4. **KimiClaw generates decision** (YES/NO/NO_TRADE with confidence)
5. **Submit via:** `python3 kimiclaw_submit.py YES 0.75 0.03 "reasoning"`
6. **Engine reads response** and saves to `llm_decisions.jsonl`
7. **Rust system executes** the trade

### No API Key Required!

Uses the existing KimiClaw connection - no external API calls needed.

### Submit Command

```bash
python3 kimiclaw_submit.py <DECISION> <CONFIDENCE> <SIZE_PCT> <REASONING>

# Examples:
python3 kimiclaw_submit.py YES 0.75 0.03 "Bullish momentum at MA7 support"
python3 kimiclaw_submit.py NO 0.60 0.02 "Bearish divergence overhead"
python3 kimiclaw_submit.py NO_TRADE 0.0 0.0 "Uncertain market conditions"
```

### Files

| File | Purpose |
|------|---------|
| `data/kimiclaw_request.json` | Auto-generated market context |
| `data/kimiclaw_response.json` | Decision submitted by KimiClaw |
| `kimiclaw_submit.py` | Submit decision tool |
| `kimiclaw_monitor.py` | Request monitor |

## Regime-Based Trading Intelligence (02:55)

### User Shared Critical Trading Insight

**BULL Regime Characteristics:**
- YES trades (green candles) outnumber NO trades **2:1**
- Huge opportunity to win on YES without heavy technical analysis
- Can be more aggressive with YES positions

**BEAR Regime Characteristics:**
- Green vs Red candles is approximately **50:50**
- Need to be more cautious
- Require **higher conviction** in technicals before committing to NO trades
- NO trades need stronger confirmation

### Trading Implications

| Regime | YES Bias | NO Bias | Strategy |
|--------|----------|---------|----------|
| **BULL** | Strong (2:1 advantage) | Avoid unless strong signal | Favor YES, lighter technical requirements |
| **BEAR** | Neutral (50:50) | Neutral (50:50) | Require strong technical conviction for either direction |

### Key Takeaway
> "In BULL regime, the base rate alone gives YES a 66% win probability (2:1 ratio). In BEAR regime, it's a coin flip - you need the technicals to give you an edge."

This is valuable context for LLM decision-making - I should weight regime heavily in the analysis.
