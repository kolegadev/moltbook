## Block #313 Investigation - COMPLETED

### Issue Found
**Slope mismatch:** Database showed -14.6° (bearish) but trade rationale claimed +30.1° (bullish)
- Actual result: YES trade placed when price went down → LOSS

### Root Cause
`auto_responder.py` regex matched slope from **14-minute history section** instead of **current metrics section**
- Prompt has multiple slope values (current + historical)
- Old regex found first match anywhere in prompt
- Historical data showed +30.1° (bullish) which was used for decision

1. Use `re.DOTALL` to match only within "CURRENT SMOOTHED METRICS" section
2. Fallback: search only in text before "14-MINUTE SMOOTHED HISTORY"
3. Added error logging with block_id for debugging

### Impact

## PIN Risk Sizing Fix - COMPLETED

### Problem
All trades were getting 50% size reduction due to elevated OBI volatility, even when:
- No pin was detected
- No manipulation risk present

### Revised Logic (March 12, 2026)
| PIN Risk | Breakout Risk | Position Size | Condition |
|----------|---------------|---------------|-----------|
| HIGH_BREAK | Any | 0% (veto) | Pin against trend |
| HIGH_HOLD | < 0.3 | 100% | Stable pin |
| HIGH_HOLD | 0.3-0.6 | 50% | **Stressed pin - ONLY 50% case** |
| HIGH_HOLD | > 0.6 | 0% (veto) | Breaking pin |
| LOW / No pin | < 0.8 | 100% | Normal volatility |
| LOW / No pin | >= 0.8 | 0% (veto) | Extreme volatility |

### Key Change
- **50% reduction ONLY for HIGH_HOLD pins with 0.3 <= breakout_risk < 0.6**
- General volatility without pin now = **Full size** or **Veto**, never 50%

### Commit
`75610b6`: fix: Revise PIN risk sizing - 50% reduction ONLY for stressed pins

- HIGH_BREAK: Always veto ✅

- HIGH_HOLD breaking: Veto ✅
- LOW no pin, low vol: Full size ✅
